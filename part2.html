ROS 2 Tutorial for Beginners:
Creating Your First Node

Understand what ROS 2 nodes are, write a Python node using rclpy, create an executable via setup.py, run it — and fix the most common mistakes beginners make.

What are Nodes?

Instead of memorising a definition, we will show you exactly what a node is by creating one. Before creating a node it is important to know how to create a workspace and a package — a node cannot exist without them.

In ROS 2, a node is a single executable program that performs a specific task. Think of a robot's software system as a city — nodes are the individual buildings, each with a purpose. One node reads the camera, another processes images, another controls the motors. Nodes communicate with each other through ROS 2's communication system.

In this tutorial, we will create a Python node that displays a message on screen. Simple — but this exact process is the foundation for every node you will ever build.

Creating a Workspace and Package

The two most popular programming languages used with ROS 2 are C++ and Python. In this tutorial we will create a Python node — Python is more beginner-friendly and an ideal starting point.

Before creating the node, create a workspace named ws and a Python package named py_pkg inside it.

First, source ROS 2:

source /opt/ros/jazzy/setup.bash
Replace jazzy with your distribution name — e.g. humble if you are using ROS 2 Humble.

Create the workspace folder:

mkdir ws

Move inside the workspace and create the source folder:

cd ws
mkdir src

Build the empty workspace:

colcon build

Now move inside the src folder and create the Python package:

cd src
ros2 pkg create py_pkg --build-type ament_python --dependencies rclpy

Move back to the workspace root and build again:

cd ..
colcon build

You should see:

Summary: 1 package finished [1.08s]
Important: colcon build must always be executed from the workspace folder, never from inside src or any package folder. The package creation command must be executed from inside the src folder.

Your workspace now has the following structure:

ws/
  src/
    py_pkg/
      py_pkg/  ← this is where your node files go
      setup.py
      package.xml
  build/
  install/
  log/

Creating the Node File

Inside the src folder we have a folder named py_pkg. Inside this package there is another folder with the same name. Navigate into it:

cd py_pkg/py_pkg/

Now create a Python file for the node:

touch my_first_node.py

Make this file executable so ROS 2 can run it later:

chmod +x my_first_node.py

Run ls to verify. The file my_first_node.py should now appear in green, indicating it is an executable.

If the file name shows in blue instead of green, the chmod command either had a syntax error or was run from the wrong directory. Run:

cd ~/ws/src/py_pkg/py_pkg/

Then run chmod +x my_first_node.py again.

Now open VS Code from the workspace root to start coding:

cd ~/ws
code .

Writing the Node Code

Open my_first_node.py in VS Code and write the following code step by step. Every line is explained below.

Step 1 — The Shebang Line

At the very top of the file, add:

#!/usr/bin/env python3

This line is called the shebang. It tells the operating system: "Use the Python 3 interpreter to execute this file." It must always be the first line.

Step 2 — Import rclpy

import rclpy

rclpy is the Python client library for ROS 2. Think of it as a folder containing Python files, and one of those files — node.py — contains the Node class we need to create a node.

Step 3 — Import the Node Class

from rclpy.node import Node

This imports the Node class from the node.py file inside the rclpy library. We need this class to create and manage a ROS 2 node.

Step 4 — Create the main() Function

def main():
    pass

All the main logic of our node goes inside this main() function. The reason will become clear when we create the executable — ROS 2 needs to know which function to call when the node runs.

Step 5 — Initialize ROS 2

Inside main(), write:

def main():
    rclpy.init()

Think of ROS 2 as a car parked in a garage. Writing import rclpy opened the garage door — you can see the car. Writing rclpy.init() starts the engine. Only after this line can you use ROS 2 functionalities.

Step 6 — Create the Node

def main():
    rclpy.init()
    node = Node("python_node")

This creates a node named python_node. The string passed to Node() is the name of your node. You can use any variable name on the left — node is just a convention.

Step 7 — Log a Message

def main():
    rclpy.init()
    node = Node("python_node")
    node.get_logger().info("I am learning ROS 2.")

This prints a message through the ROS 2 logging system. Replace "I am learning ROS 2." with whatever message you want to display.

Step 8 — Shut Down ROS 2

def main():
    rclpy.init()
    node = Node("python_node")
    node.get_logger().info("I am learning ROS 2.")
    rclpy.shutdown()

We started the ROS 2 engine with rclpy.init(). Now we turn it off with rclpy.shutdown(). Always close what you open.

Complete Node Code

Your complete my_first_node.py should look like this:

#!/usr/bin/env python3

import rclpy
from rclpy.node import Node

def main():
    rclpy.init()
    node = Node("python_node")
    node.get_logger().info("I am learning ROS 2.")
    rclpy.shutdown()

Press Ctrl + S to save the file.

Creating an Executable

In ROS 2, finishing the code is not enough — you must register an executable so ROS 2 knows how to run your node.

In VS Code's left sidebar, locate setup.py inside your package (ws/src/py_pkg/setup.py). Open it.

Find the console_scripts section and add your executable entry inside the square brackets:

entry_points={
    'console_scripts': [
        'my_node = py_pkg.my_first_node:main',
    ],
},

Breaking this down:

  • my_node — the name of the executable (what you will type to run the node)
  • py_pkg — the name of the package
  • my_first_node — the name of the Python file (without .py)
  • main — the function to call when the executable runs

This is why we wrote our code inside main() — the executable entry point must reference a function name.

Save the file after making this change (Ctrl + S). A common mistake is running colcon build before saving, which means the executable registration is never included in the build.

Now go to your terminal, navigate to the workspace root, and build:

cd ~/ws
colcon build

After building, source the install folder:

source ~/ws/install/setup.bash

Running the Node

Run the node using:

ros2 run py_pkg my_node

The format is: ros2 run <package_name> <executable_name>. Note that you type the executable name (my_node), not the node name (python_node).

If everything is set up correctly, you will see:

[INFO] [python_node]: I am learning ROS 2.

Your first ROS 2 node is working.

Common Mistakes To Avoid

If your node did not run and you saw an error instead, you likely made one of the following mistakes. Go through each one carefully.

Mistake 1 Running colcon build from the Wrong Directory

If you ran colcon build from inside src or a package folder, the build went to the wrong place. Check for build, install, and log folders in that wrong directory. If they exist, delete them:

rm -rf build install log

Then go to your workspace and build correctly:

cd ~/ws
colcon build
Mistake 2 Not Sourcing the Install Folder

If you get an error like Package 'py_pkg' not found when trying to run your node, you have not sourced the install folder. Run these two commands:

source /opt/ros/jazzy/setup.bash
source ~/ws/install/setup.bash

Replace jazzy with your ROS 2 distribution. Now try running the node again.

Mistake 3 Forgetting to Save Files

We made changes to two files: my_first_node.py and setup.py. If either was not saved before running colcon build, the unsaved changes were not included in the build. Always press Ctrl + S to save both files, then rebuild:

colcon build
source ~/ws/install/setup.bash
ros2 run py_pkg my_node
Mistake 4 Syntax Errors While Typing Commands

ROS 2 commands are case-sensitive and space-sensitive. A single extra space or wrong letter will cause an error. Compare your commands letter-by-letter against the ones in this tutorial if something is not working.

You can also compare your file structure against the reference repository: github.com/Satyarth-Shree/ros2-beginner-part-2-python-node

Conclusion

You have just written, built, and run your first ROS 2 node. This is the most fundamental skill in ROS 2 development — every robot program you write from here will follow this same pattern: create a file, write a node class, register an executable, build, and run.

Practice this by creating multiple workspaces and nodes from scratch until the process feels natural. In the next part of this series, we will go deeper — exploring what nodes truly are, how to run multiple nodes simultaneously, and how to introspect them using ROS 2 command-line tools.
