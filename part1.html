ROS 2 Tutorial for Beginners:
Create Your First Workspace and Package

A step-by-step guide covering ROS 2 distributions, installation on Ubuntu 24.04, essential Linux terminal commands, workspace creation, and your first Python package.

This tutorial has been made for beginners who are just starting with ROS 2 and are entirely unaware of its features and functionalities. Let's dive in and explore what ROS 2 is and how it works.

💡 Tip: Use split-screen — keep this guide on one side and your terminal on the other. If you have a phone, open this guide there and follow along on your laptop or PC.

ROS 2 Distributions

First, open this link: https://docs.ros.org/en/foxy/Releases.html

After scrolling down you will find a list of distributions:

List of ROS 2 Distributions

The distributions highlighted green are currently supported by the ROS 2 manufacturers. Some are supported for 5 years while others for shorter durations — for example, Iron Irwini has an EOL of November 2024, while Humble Hawksbill is supported for 5 years. Always choose long-term supported distributions.

Future ROS 2 Distributions

Under "Future Distributions," Jazzy Jalisco is visible — released in 2024 with EOL in 2029. It is the most recent long-term supported ROS 2 distribution. We will use Jazzy in this tutorial.

If you're reading this several years after publication, install the most recently supported distribution of that time. The core concepts covered here remain identical across distributions.

Installing ROS 2 Jazzy

Search "ROS2 Jazzy Documentation" in your browser and click the first official result:

Search result for ROS 2 Jazzy Documentation

ROS 2 Official Documentation Page

Click Installation. Use Ubuntu Linux 24.04 — only this version is compatible with ROS 2 Jazzy and it provides the best support and documentation.

ROS 2 Installation Options

Under Ubuntu Linux → deb packages, scroll down to find the installation commands:

Ubuntu deb packages installation page

Commands for ROS 2 Installation

Copy-paste these commands one by one into your terminal. Then install development tools (recommended):

Install development tools

Choose Desktop Install (not Base Install) — this gives you simulation tools as well:

Desktop Install vs Base Install

After all commands complete successfully, verify the installation:

source /opt/ros/jazzy/setup.bash
ros2

If a list of ROS 2 commands appears, installation was successful. If you see ros2 command not found, something went wrong.

Now install two additional tools — VS Code and gedit:

sudo snap install code --classic
sudo apt install gedit

Verify gedit by running gedit — a text editor window should open:

Gedit text editor — successfully installed

Understanding Basic Linux Terminal Commands

If you already know cd, ls, and mkdir, you can skip to the next section.

Creating Folders — mkdir

Create a folder named my_first_folder:

mkdir my_first_folder

Open the Files app to confirm — the folder should appear in your home directory:

my_first_folder created in home directory

When naming folders, avoid spaces. Use underscores to separate words: my_first_folder, not my first folder.

Changing Directory — cd

To create a folder inside my_first_folder, change your working directory into it first:

cd my_first_folder/

Terminal after changing directory — my_first_folder is now active

Now create folder_1 inside it:

mkdir folder_1

folder_1 created inside my_first_folder

To go one level back to the home directory:

cd ..

Back to the root/home directory

Now create my_second_folder in the home directory:

mkdir my_second_folder

my_second_folder created successfully

Visualise the folder structure like this:

Folder hierarchy visualised

Listing Files — ls

The ls command lists all files and folders in the current working directory:

ls

ls showing home directory contents

ls inside my_first_folder

ls inside my_second_folder — empty since nothing was created inside it

Creating a Workspace in ROS 2

Setting Up the ROS 2 Environment

Before using ROS 2, its environment must be sourced. Open a new terminal and type:

source /opt/ros/jazzy/setup.bash

To avoid running this every session, add it to .bashrc so it runs automatically:

gedit .bashrc

Opening the .bashrc file in gedit

Scroll to the bottom of the file and add the source command as shown:

Adding the ROS 2 source command at the end of .bashrc

Save and close the file. Now open a new terminal — ROS 2 will be ready without manually sourcing it each time.

Creating the Workspace

First install colcon (the ROS 2 build tool):

sudo apt update
sudo apt install python3-colcon-common-extensions

Create the workspace folder named ws_1, move inside it, and create the source folder:

mkdir ws_1
cd ws_1
mkdir src

Use ls to confirm the src folder exists:

src folder successfully created inside ws_1

Now build the workspace:

colcon build

Successful colcon build output

Run ls again — three new folders will appear: build, install, and log:

build, install, and log folders created after colcon build

Add the workspace source to .bashrc as well so your custom nodes are always available:

source ~/ws_1/install/setup.bash

Go back to the home directory first with cd .., then open .bashrc with gedit .bashrc, add the line above at the end, save and close.

Creating a Package in ROS 2

Packages must be created inside the src folder. Navigate there:

cd ws_1/src/

Empty src folder — ready for the first package

ROS 2 has two types of packages: Python packages and C++ packages. We will create a Python package. Create one named my_first_package:

ros2 pkg create my_first_package --build-type ament_python --dependencies rclpy
For a C++ package instead, use:
ros2 pkg create my_first_package --build-type ament_cmake --dependencies rclcpp

Output after running the package creation command

Use ls to confirm the package folder exists:

my_first_package visible inside src

Move back to the workspace root and build:

cd ..
colcon build

Build output — 1 package finished

You should see:

Summary: 1 package finished [0.94s]

Navigate inside the package to verify it was created correctly:

Files automatically generated inside the package — creation confirmed

Auto-generated files and folders appearing inside your package confirm it was built successfully. You are now ready for Part 2 — creating your first node inside this package.
